# Peak Time Monitor - Project Report

## 1. Overview

The Peak Time Monitor (PTM) is a digital ASIC design that continuously
samples an 8-bit input bus and tracks two values:

- **peak** - the highest value seen so far
- **interval** - the number of clock cycles between the previous peak
  and the new peak

Sampling occurs once every `DIV_RATIO` system clock cycles, controlled
by an internal clock-enable pulse generated by a clock divider
submodule. The design was carried through the complete RTL-to-GDSII
digital implementation flow on the SCL180 PDK.

## 2. Design Flow Summary

- **Simulation** (Vivado / ModelSim) – RTL functionally verified against a self-checking reference model
- **Synthesis** (Cadence Genus) – RTL mapped to SCL180 standard cells under a 30 ns clock constraint
- **Floorplan** (Cadence Innovus) – 1940 x 1940 micron die, 125 micron core-to-I/O margin
- **Powerplan** (Cadence Innovus) – VDD/VSS rings and stripes distributing supply across the core
- **Placement** (Cadence Innovus) – Legalized cell placement, congestion-checked, pre-CTS timing closed
- **Clock Tree Synthesis** (Cadence Innovus, CCOpt) – Clock tree built to a 0.5 ns target skew with useful-skew optimization
- **Routing** (Cadence Innovus) – Timing- and SI-driven global/detail routing with antenna repair
- **Signoff** (Cadence Innovus) – Final STA, DRC/connectivity checks, and GDSII streamout

## 3. Key Learnings

- End-to-end exposure to the standard digital ASIC implementation
  flow, from RTL simulation through GDSII generation.
- Practical experience writing SDC constraints and observing their
  direct effect on synthesis and timing closure.
- Hands-on understanding of floorplanning, power distribution network
  design, placement congestion management, clock tree synthesis with
  useful skew, and signoff-quality verification.
- Appreciation for how each stage's output feeds as a constraint or
  input into the next stage of the flow.
